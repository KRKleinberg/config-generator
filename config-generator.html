Guide

Config Generator Guide

This tool is designed to generate error-free, standardized network configurations entirely locally within your browser. No data is transmitted over the network. No data persists in the browser after this tab is closed.

Workspace & Files

  • Template File: Click Load under Template to upload a Jinja-formatted configuration file. Note: If your template does not contain !!! Section Name markers, the preview will automatically switch to Full mode.
  • Variables File (Optional): Click Load under Variables to upload an INI file to define the form structure, descriptions, dropdowns, and regex validation.
  • Auto-Detect : Click the lightning bolt icon next to the progress bar to instantly generate missing form fields by scanning your template.
  • Export INI : If you use Auto-Detect without a variables file loaded, a download icon appears next to it. Click it to generate a boilerplate .vars.ini schema file for template maintainers.

Generating Configurations

  • Fill out the form fields on the left. The Preview panel on the right will update in real-time.
  • Click the Reset icon above the form to clear all your typed inputs without removing your loaded files.
  • Fields marked with a * are required and can be dynamically updated based on which fields are filled.
  • Click any highlighted {{ VARIABLE }} in the Preview panel to automatically scroll to and flash its corresponding input field on the left.

Manual Editing

If a specific deployment requires a unique command not covered by the template, you can manually edit the output.

  • Section Mode: Click on any specific section header to edit just that block of text.
  • Full Mode: Toggle the view to Full at the top right, then click to edit the entire configuration as a single text file.
  • If you edit a section, a yellow (edited) badge will appear. To revert to the template's automated output, click the reset button on that specific section.
  • After editing, you can save the edited template by clicking the Save Template button in the top right.

Exporting & Sessions

  • Copying: Click to copy a section or the full config to your clipboard. Badges (e.g., !STIG: V-123456 ) are automatically stripped from the copied text.
  • Saving: Click Save Config in the top right to save the filled in config to your files.
  • Sessions: Use the Save Session icon (top left) to save your current progress as a .session.json file. This saves your loaded templates, variables, form inputs, and manual edits so you can safely close your browser and resume work later using the Load icon.

Limitations

  • INI Constraints: Variables are defined as flat [VAR_NAME] sections of key = value lines. Nested structures are not supported, and choices must be an inline list ( [opt1, opt2] ). Unrecognized keys are ignored with a warning.
  • Jinja Constraints: Conditionals only support existence checks (e.g., {% if VARIABLE %} ). They do not support equality checks (e.g., {% if VARIABLE == 'value' %} ).
  • Variable Names: Must be alphanumeric with underscores/hyphens. Spaces in variable names will break the regex.
